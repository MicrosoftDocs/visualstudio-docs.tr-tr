# <a name="c6262"></a>C6262
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Uyarı C6262: Işlev \<constant> yığının baytlarını kullanıyor:/Analyze: stacksize değerini aşıyor \<constant 2> . Bazı verileri yığına taşımayı düşünün  
  
 Bu uyarı, bir işlevde önceden ayarlanmış eşiği () aşan yığın kullanımının `constant 2` algılandığını gösterir. Bu uyarı için varsayılan yığın çerçeve boyutu, Kullanıcı modu için 16 KB, çekirdek modu için 1 KB 'tır. Yığın — Kullanıcı modunda bile — sınırlıdır ve yığın taşması özel durumuna neden olan yığının bir sayfasını yürütmek için başarısız olur. Çekirdek modunun 12 KB 'lik yığın boyutu sınırı vardır ve bu artış artamaz; Bu nedenle, çekirdek modu kodu, yığın kullanımını kararlılık olarak sınırlandırmalıdır.  
  
 Bu uyarının arkasındaki sorunu gidermek için, bazı verileri yığına ya da diğer dinamik belleğe taşıyabilirsiniz.  Kullanıcı modunda, büyük bir yığın çerçevesi bir sorun olmayabilir ve bu uyarı bastırılabilir, ancak büyük bir yığın çerçevesi yığın taşması riskini artırır. (İşlev yığını yoğun veya yinelemeli olarak kullanıyorsa büyük bir yığın çerçevesi meydana gelebilir.) Yığın taşması gerçekleşirse, ancak yalnızca sistem sınırına kadar, kullanıcı modundaki toplam yığın boyutu artırılabilir.  Değerini değiştirmek için **/Analyze** komut satırı seçeneğini kullanabilirsiniz `<constant 2>` , ancak bunu artırmak bir hatanın bildirilmeyecek bir riski ortaya çıkarır.  
  
 Çekirdek modu kodu için — örneğin, sürücü projelerinde — değeri `<constant 2>` 1 KB olarak ayarlanır. İyi yazılmış sürücülerde, bu değere yaklaşılarak çok az sayıda işlev olmalıdır ve sınırı aşağı doğru olarak değiştirmeniz istenebilir.  Yığın boyutunu azaltmak için Kullanıcı modu kodu için kullanılan aynı genel tekniklerin çekirdek modu koduna uyarlanabilirler.  
  
## <a name="example"></a>Örnek  
 Aşağıdaki kod bu uyarıyı oluşturur çünkü `char buffer` 16.382 bayt ayırır ve yerel tamsayı değişkeni `i` , bırlıkte 16 KB varsayılan yığın boyutu sınırını aşan başka 4 bayt ayırır.  
  
```cpp  
// cl.exe /c /analyze /EHsc /W4  
#include <windows.h>  
#define MAX_SIZE 16382  
  
void f( )  
{  
    int i;  
    char buffer[MAX_SIZE];  
  
    i = 0;  
    buffer[0]='\0';  
  
    // code...  
}  
  
```  
  
 Aşağıdaki kod, bazı verileri yığına taşıyarak bu uyarıyı düzeltir.  
  
```cpp  
// cl.exe /c /analyze /EHsc /W4  
#include <stdlib.h>     
#include <malloc.h>  
#define MAX_SIZE 16382  
  
void f( )  
{  
    int i;  
    char *buffer;  
  
    i = 0;  
    buffer = (char *) malloc( MAX_SIZE );  
    if (buffer != NULL)   
    {  
        buffer[0] = '\0';  
        // code...  
        free(buffer);  
    }  
}  
  
```  
  
 Malloc ve Free kullanımı, bellek sızıntıları ve özel durumlar bakımından birçok ekde sahiptir. Bu tür sızıntıları ve özel durum sorunlarını tamamen önlemek için, C++ Standart Şablon kitaplığı (STL) tarafından sunulan mekanizmaları kullanın. Bunlar arasında [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)ve [vektör](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)bulunur. Daha fazla bilgi için bkz. [akıllı işaretçiler](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) ve [C++ standart kitaplığı](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).  
  
 **Yığın boyutunu ayarlayarak bu uyarıyı düzeltmek için**  
  
1. Menü çubuğunda, **Proje**, **Özellikler**' i seçin.  
  
     **Özellik Sayfaları** iletişim kutusu görüntülenir.  
  
2. **Yapılandırma özellikleri**' ni genişletin.  
  
3. **C/C++**' ı genişletin.  
  
4. **Komut satırı** özellikleri ' ni seçin.  
  
5. **Ek seçenekler**' de, **/Analyze: stacksize16388**ekleyin.
