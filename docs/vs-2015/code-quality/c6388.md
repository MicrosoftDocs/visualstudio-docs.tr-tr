# <a name="c6388"></a>C6388
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Uyarı C6388: \<argument> Bu durum, \<value> işlevin belirtimine uymuyor: \<function name> satırlar: x, y  
  
 Bu uyarı, belirtilen bağlamda beklenmeyen bir değerin kullanıldığını gösterir. Bu, genellikle, beklenmeyen bir işleve bağımsız değişken olarak geçirilen değerler için raporlanır.  
  
## <a name="example"></a>Örnek  
 Şu C++ kodu bu uyarıyı oluşturur çünkü Dosomeu null değer bekliyordu, ancak null olmayan bir değer geçirilebilir:  
  
```cpp  
  
#include <string.h>  
#include <malloc.h>  
#include <sal.h>  
  
void DoSomething( _Pre_ _Null_ void* pReserved );  
  
void f()  
{  
    void* p = malloc( 10 );  
    DoSomething( p );  // Warning C6388  
    // code...  
    free(p);  
}  
```  
  
 Bu uyarıyı düzeltmek için aşağıdaki örnek kodu kullanın:  
  
```cpp  
  
#include <string.h>  
#include <malloc.h>  
#include <sal.h>  
  
void DoSomething( _Pre_ _Null_ void* pReserved );  
void f()  
{  
    void* p = malloc( 10 );  
    if (!p)  
    {  
        DoSomething( p );    
    }  
    else  
    {  
        // code...  
        free(p);  
    }  
}  
```  
  
 Malloc ve Free 'in kullanılması, bellek sızıntıları ve özel durumlar açısından birçok Bu tür sızıntıları ve özel durum sorunlarını tamamen önlemek için, C++ Standart Şablon kitaplığı (STL) tarafından sunulan mekanizmaları kullanın. Bunlar arasında [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)ve [vektör](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)bulunur.
